# Week 3 Assignment: Automated Regional Impact Auditor (ARIA)

## 專案概述

本專案為第三週作業：河川洪災避難所風險評估系統。利用水利署河川圖資建立多級警戒緩衝區，結合消防署避難收容所資料，評估各行政區的避難所洪災風險與收容量缺口。

## 資料來源

- **河川資料**：水利署河川面（Polygon）- EPSG:3826
- **避難收容所資料**：消防署（政府開放平台）- 5,907 筆記錄
- **鄉鎮市區界**：國土測繪中心 - 368 個行政區

## AI 診斷日誌 (AI Diagnostic Log)

### 遇到的問題與解決方案

#### 1. 資料清理問題
**問題**：原始避難所資料包含異常座標（經緯度為 0 或超出台灣範圍）
**解決方案**：設定合理的座標範圍（經度：119-122.5，緯度：21-26.5）進行過濾，移除 66 筆異常記錄

#### 2. 幾何圖形修復
**問題**：河川資料中存在 1 個無效的幾何圖形
**解決方案**：使用 `buffer(0)` 方法修復無效幾何圖形

#### 3. 空間結合匹配問題
**問題**：30 個避難所無法匹配到行政區
**解決方案**：使用 `how='left'` 參數保留所有記錄，並在分析中處理未匹配的記錄

#### 4. 聯集操作效能問題
**問題**：大量河川幾何圖形的聯集操作可能導致效能問題
**解決方案**：使用 `union_all()` 方法提升效能，並提供備用的分別處理方案

#### 5. 中文字型顯示
**問題**：matplotlib 中文顯示異常
**解決方案**：設定多種中文字型作為備選：`['Microsoft JhengHei', 'SimHei', 'Arial Unicode MS']`

#### 6. JSON 匯出格式
**問題**：確保 JSON 支援中文顯示
**解決方案**：使用 `force_ascii=False` 參數，確保中文字元正確顯示

### 學習心得

1. **資料清理的重要性**：真實世界的政府開放資料往往包含異常值，需要仔細驗證和清理
2. **座標系統的一致性**：所有空間分析必須在相同的座標系統下進行（EPSG:3826）
3. **緩衝區分析的邏輯**：多級緩衝區需要正確的覆寫邏輯，避難所落在多個緩衝區時應取最高風險等級
4. **視覺化的多樣性**：同時提供靜態圖表和互動式地圖，滿足不同使用者的需求
5. **錯誤處理的必要性**：地理空間分析中常有意外情況，需要完善的錯誤處理機制
